Jupiter API Key — Why You Need One

▼
What is it?

Jupiter is the leading DEX aggregator on Solana. The bots use Jupiter's API to fetch real-time token prices and execute swaps. A free API key gives you authenticated access to higher rate limits and ensures your bot can trade without being throttled or blocked.

Why is it required?
  • The Jupiter swap and quote endpoints require an API key for all requests
  • Without a key, price fetches and swap attempts will return 401 Unauthorized errors and your bot will not trade
  • The key is free — no credit card required
  • Your key is written only to your downloaded .env file and never sent to our servers
1
Visit the Jupiter Developer Portal
Go to portal.jup.ag and create a free account. No payment details needed.
2
Create a new API key
In the portal dashboard, click Create API Key. Give it a label (e.g. "trading bot") and copy the key — it looks like a long alphanumeric string.
3
Enter it in the Bot Builder
On the Bot Builder page, paste your key into the Jupiter API Key field. It will be saved in your browser and written to your downloaded .env file.
4
Enter it on the Charts page (optional)
Click 🔑 Connect in the top nav of the Charts page to enable live price data. Uses the same key.
🛡️
Your key stays private
Your API key is stored only in your browser's localStorage and written to your local .env file when you download a bot. It is never transmitted to or stored on our servers.

Installing Python 3 & VS Code

▼
Requirements
  • Python 3.10 or higher — the bots use modern syntax and async features not available in older versions. Python 3.11 or 3.12 is recommended.
  • pip3 — the Python package installer (comes bundled with Python 3.4+)
  • VS Code — recommended editor for editing your .env config and monitoring bot output

macOS

1
Install Homebrew (if not already installed)
Open Terminal and run:
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
2
Install Python 3
Run: brew install python@3.12 — then verify with python3 --version. You should see Python 3.12.x.
3
Install VS Code
Download from code.visualstudio.com or run brew install --cask visual-studio-code

Windows

1
Download Python 3 from python.org
Go to python.org/downloads/windows and download the latest Python 3.12 installer. Run it and check "Add Python to PATH" during install.
2
Verify the install
Open Command Prompt or PowerShell and run python --version. You should see Python 3.12.x.
3
Install VS Code
Download from code.visualstudio.com and run the installer.

Linux (Ubuntu / Debian)

1
Install Python 3
Run: sudo apt update && sudo apt install python3 python3-pip — verify with python3 --version
2
Install VS Code
Run: sudo snap install --classic code or download the .deb package from code.visualstudio.com

Deploying Your Bot

▼

Once you've built and downloaded a bot from the Bot Builder, follow these steps to get it running on your machine.

1
Unzip the downloaded folder
Extract solana-[strategy]-bot.zip to a folder on your computer. You'll see bot.py, .env, requirements.txt, and run.sh.
2
Review and edit the .env file
Open .env in VS Code. Verify your private key, cashout address, RPC URL, Jupiter API key, and strategy settings. Never share this file with anyone.
3
Install dependencies
Open a terminal in the bot folder and run: pip3 install -r requirements.txt
Or just run bash run.sh which handles this automatically.
4
Start the bot
Run: python3 bot.py
The bot starts a WebSocket server on port 8080. You'll see startup logs including price baseline and watch list.
5
Connect the live terminal
Go to the Bot Builder or Charts page, scroll to the Bot Terminal section, type localhost:8080 and click Connect. You'll see live logs and can type commands like status, pause, stop.
6
Run dry run first
In your .env set DRYRUN=true and run the bot. It will simulate all trades without spending real USDC. Watch for price fetches, entry signals, and exit logic. Only switch to DRYRUN=false when you're satisfied.
💡
Keeping the bot running 24/7
To run your bot continuously on a server or your own machine while you sleep, consider using a VPS (like DigitalOcean, Hetzner, or Vultr), or running the bot in a screen or tmux session on macOS/Linux so it stays alive after you close the terminal.

DCA Bot
Dollar Cost Average — automated recurring buys
▼
What it does

The DCA bot buys a fixed USDC amount of a target token on a set schedule — every 5 minutes, hourly, daily, or custom. It tracks your average entry price and can automatically stop-loss and cashout if your position falls below a threshold. Ideal for slowly accumulating SOL, JUP, or any Solana token over time without timing the market.

When to use it
  • You believe in a token long-term but don't want to buy all at once
  • You want to remove emotion from buying decisions
  • You have a fixed budget you want to deploy gradually
  • Accumulating during sideways or bearish markets
Recommended starter settings
SettingRecommendedNotes
BUYAMOUNT$5–$20 USDCSmall enough to run many buys without depleting pool
INTERVAL1h or 4hMore frequent = more fees. 1h is a good balance.
BUDGETCAPEqual to your poolBot stops when total spent hits this
STOPLOSS20–30%Sell all holdings if down this % from average entry
SLIPPAGEBPS500.5% — fine for major tokens
DRYRUNtrue (first)Always test first
📊
Risk level
LOW-MED   DCA reduces timing risk but doesn't eliminate market risk. A falling token will keep being bought until the budget cap or stop-loss fires.

Copy Bot
Mirror a wallet's swaps in real time
▼
What it does

The Copy bot monitors a target Solana wallet address for new transactions. When it detects a swap, it mirrors that trade using your own bot pool funds. You set a fixed USDC amount per copied trade, a max drawdown limit, and a blacklist of tokens you never want to copy. Uses RPC polling to watch for new transactions.

When to use it
  • You've found a consistently profitable on-chain trader
  • You want to follow a wallet associated with insider knowledge
  • You'd rather piggyback on someone else's research
  • You want exposure to a style of trading without learning it yourself
Recommended starter settings
SettingRecommendedNotes
POSITIONSIZE$10–$25 USDCKeep small — you don't know what you're copying
POLLINTERVAL10sFaster polling = more RPC calls. 5s only with private RPC.
MAXDRAWDOWN15–20%Pauses bot if pool drops this % from peak
SLIPPAGEBPS1001% — copy trades need to fill fast
MAXPOSITIONS3–5Limit concurrent exposure
⚠️
Copy trading risk
Past performance of a wallet does not guarantee future results. The wallet you're copying could be a market maker, a bot itself, or simply get lucky. Never copy a wallet you haven't monitored manually for at least a week. Your bot will always be slightly behind the target due to polling delay.

Momentum Bot
Enter breakouts — ride the move, exit at target
▼
What it does

Scans a watch list of tokens every scan interval. When a token gains more than your threshold % in one scan window, the bot enters a position with a fixed USDC amount. It then watches that position for a take-profit or stop-loss trigger, and exits accordingly. The bot also enforces a daily loss cap and max concurrent positions limit.

When to use it
  • You want to catch momentum moves on trending tokens
  • You're comfortable with more frequent entries and exits
  • Market is generally trending up (green days)
  • You have a curated list of liquid, volatile tokens to watch
Recommended starter settings
SettingRecommendedNotes
GAINTHRESHOLD3–5%Lower = more entries but more false positives
SCANINTERVAL5mGood balance of signal quality vs RPC calls
POSITIONSIZE$15–$30 USDCKeep to ~10% of pool per position
TAKEPROFIT6–10%Higher TP needs stronger trend
STOPLOSS3–5%Keep tighter than TP for positive risk/reward
MAXPOSITIONS3Start small — 3 concurrent is plenty
📊
Risk level
MEDIUM   Momentum trading works well in trending markets and poorly in choppy sideways markets. The daily loss cap is your emergency brake — set it seriously.

Scalper Bot
Micro-moves — 0.3% entries, tight exits, high frequency
▼
What it does

The most active bot. Scans every 10–30 seconds, enters on tiny price moves (0.1–0.3%), and exits quickly at very small take-profit or stop-loss levels. Uses DexScreener for real-time pricing and auto-refreshes its watch list from trending tokens. Enforces a daily trade cap and daily loss cap to prevent runaway losses. Only trades tokens that are green on the daily chart (configurable).

When to use it
  • You want high activity and many small wins
  • Market is liquid and trending (avoid during low volume)
  • You have a private RPC — public RPCs are too slow for scalping
  • You want the bot to self-manage its token selection
  • You understand that small wins can be wiped by fees
Recommended starter settings
SettingRecommendedNotes
GAINTHRESHOLD0.1–0.3%Lower threshold = more entries, more fees
SCANINTERVAL30s10s only with a fast private RPC
POSITIONSIZE$10–$20 USDCSmall — bot opens many positions
TAKEPROFIT0.3–0.5%Must exceed Jupiter swap fee (~0.1%)
STOPLOSS0.2–0.3%Keep tight — don't let small scalps become big losses
DAILYTRADECAP100–200Prevents fee bleed if market is choppy
VERIFIEDONLYtrueStrongly recommended — avoid rug pulls
⚡
Scalping requires a private RPC
Public RPC endpoints (api.mainnet-beta.solana.com) have rate limits and latency that will cause missed entries and failed transactions. For serious scalping, use a private RPC from Helius, QuickNode, or Alchemy. Set it as RPCURL in your .env.
📊
Risk level
HIGH   Scalping is the most demanding strategy. Slippage, fees, and market noise can easily turn small profits into losses. Run dry mode for at least 24 hours before going live.

Dip Buyer
Buy the dip, DCA down, sell the recovery
▼
What it does

Watches a set of tokens and enters a position when the hourly chart drops by a set percentage. If price continues falling, it DCA's down by buying more at each additional step drop — up to a maximum number of buys. When price recovers above your average entry by the take-profit %, it sells everything back to USDC. In Long Mode, it never stop-losses — it just keeps accumulating and waits for the recovery.

When to use it
  • You believe in a token long-term but want to buy weakness not strength
  • You want to accumulate SOL or blue-chip Solana tokens on dips
  • You want automated dip-buying without watching charts manually
  • Long Mode: for tokens you'd hold regardless — bot just keeps buying dips
  • You want a strategy that benefits from volatility rather than fighting it
Recommended starter settings
SettingRecommendedNotes
DIPTHRESHOLD3–5%Enter when token drops this % on the hourly chart
POSITIONSIZE$15–$25 USDCPer buy — including DCA-down buys
DCASTEP3–5%Buy more each time price drops this % below last buy
MAXBUYSPTOKEN3–4Caps total exposure per token
TAKEPROFIT8–15%Sell all when price recovers this % above avg entry
STOPLOSS20% or blankLeave blank to enable Long Mode — no stop-loss ever
LONGMODEtrue (for SOL/JUP)Never sells at a loss — holds until take-profit fires
SCANINTERVAL5mChecks hourly dex change each scan
💡
Long Mode explained
When Long Mode is on (or Stop Loss is left blank), the bot never sells at a loss — it only exits at take-profit. This means if a token keeps falling, the bot keeps buying down until it hits MAX_BUYS, then holds until recovery. This is intentional — it's designed for tokens you'd hold anyway. Do not use Long Mode on low-conviction or unverified tokens.
📊
Risk level
MEDIUM   Risk depends heavily on token selection and Long Mode. With stop-loss on liquid verified tokens it's moderate. In Long Mode on volatile tokens it can hold losing positions for extended periods.
⚠️
Long Mode + unverified tokens = high risk
Never enable Long Mode on unverified or low-liquidity tokens. If a token rugs or goes to zero in Long Mode, the bot holds and never exits. Enable "Verified tokens only" whenever using Long Mode.
⚠️

Risks, Safety & Security

💸
Trading bots can and do lose money
All trading carries risk. Automated bots can lose money faster than manual trading because they operate continuously and react to every price move. Never fund a bot with more USDC than you are prepared to lose entirely. Start with the minimum ($10–$20) and scale up only after sustained dry-run success.
🦠
Unverified token risks
Unverified tokens on Solana can be rug pulls, honeypots, or scams with no liquidity. The bot includes a "Verified tokens only" setting that filters entries to Jupiter-verified tokens only. Always enable this setting unless you have a specific curated list of tokens you've researched manually. Trending tokens from DexScreener are filtered by liquidity and volume, but no filter is perfect.
🔐
Never share your private key or .env file
Your bot wallet private key is stored in your .env file. Anyone who gets this file has full control of the funds in that wallet. Never upload it to GitHub, Discord, Telegram, or paste it anywhere online. Do not store it in cloud sync folders (Dropbox, iCloud, Google Drive) unencrypted.
🔑
Use a dedicated bot wallet — not your main wallet
The bot builder generates a fresh keypair for your bot pool. This is intentional. Never use your primary Phantom or Solflare wallet as the bot wallet. Keep your main holdings separate. Only send the USDC you want the bot to trade into the bot wallet — nothing else.
✅
Best practices summary
Always run dry mode first for at least 24 hours  ·  Enable "Verified tokens only" for the scalper  ·  Set a realistic daily loss cap  ·  Use a dedicated bot wallet with only the funds you want to trade  ·  Keep your .env file private  ·  Monitor your bot daily via the live terminal  ·  Never invest more than you can afford to lose
